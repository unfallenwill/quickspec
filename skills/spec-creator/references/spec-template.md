## 使用说明

- 替换所有 `<占位符>` 为实际值。
- 删除不适用的章节（如功能仅涉及前端，则删除 API 接口章节）。
- 根据需要添加章节（如数据库变更需要添加"迁移计划"章节）。
- 保持每个任务聚焦 — 如果一个任务需要超过 10 行细节，将其拆分为子任务。
- 生成的规格文档应该是自包含的 — 编码 Agent 可以直接执行，无需回溯 PRD。
