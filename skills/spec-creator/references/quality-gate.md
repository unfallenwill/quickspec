# 质量门禁：生成规格文档

## 产出物

`{workspace}/{feature-name}-spec.md`

## 目标

产出一份自包含的、编码 Agent 可无歧义执行的实现规格文档。Agent 读完此文档后无需回溯 PRD 即可开始编码。

## 使用场景

| 消费者 | 消费什么 |
|--------|----------|
| Step 5 (质量审查) | `{workspace}/{feature-name}-spec.md`，用于 PRD 覆盖度交叉对照和任务可操作性检查 |
| Step 6 (输出交付) | `{workspace}/{feature-name}-spec.md`，最终交付物，直接输出给用户或 coding agent |

## 验收标准

### 文件存在性

- [ ] `{workspace}/{feature-name}-spec.md` 文件存在且可读
- [ ] 文件非空（内容长度 > 0）
- [ ] 文件结构正确（包含任务、数据模型、接口、约束、验收条件等章节）

### 任务质量

- [ ] 每个任务有精确的文件路径（非模糊目录）
- [ ] 每个任务有明确的操作类型（create / modify / delete）
- [ ] 每个任务有详细的行为描述（编码 Agent 可直接执行）
- [ ] 每个任务有错误处理定义
- [ ] 任务间的依赖关系已标注
- [ ] 任务按正确顺序排列（数据模型 → 核心逻辑 → UI）

### 数据模型与接口

- [ ] 所有实体字段有类型定义
- [ ] 字段校验规则已指定
- [ ] 实体间关联关系已明确
- [ ] API 接口有完整的请求参数（含位置、类型、必填）
- [ ] API 接口有成功响应结构
- [ ] API 接口有错误响应覆盖（至少 400/404/500）

### 约束条件

- [ ] "必须遵循"列出了需复制的现有模式
- [ ] "不能改动"列出了超出范围的文件（含原因）
- [ ] "必须复用"列出了需利用的现有工具（含文件路径）
- [ ] 性能约束已指定（如 PRD 中有）
- [ ] 安全约束已指定

### 验收条件

- [ ] 每个功能（F1, F2...）都有对应的验收条件
- [ ] 每条验收条件是布尔型（通过/不通过）
- [ ] 每条验收条件有验证方式
- [ ] 每条验收条件有测试类型标注
- [ ] 覆盖正常路径、错误路径和边界情况

### 自包含性

- [ ] spec 文档不依赖 PRD 原文即可被理解
- [ ] 所有术语在文档内有定义或已被代码库上下文覆盖
- [ ] "范围外"章节存在且明确了排除项
