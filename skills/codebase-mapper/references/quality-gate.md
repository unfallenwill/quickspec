# 质量门禁：代码库上下文映射

## 产出物

`{workspace}/codebase-mapping.md`

## 目标

映射 PRD 需求到现有代码库，确定新代码应添加的位置、应遵循的模式、应复用的组件，为规格文档生成提供精确的实现上下文。

## 使用场景

| 消费者 | 消费什么 |
|--------|----------|
| Step 4 (生成规格) | `{workspace}/codebase-mapping.md` 中的技术栈、目录结构、相关模块、约定、扩展点、依赖关系 — 用于生成精确的文件路径和遵循模式 |

## 验收标准

### 文件存在性

- [ ] `{workspace}/codebase-mapping.md` 文件存在且可读
- [ ] 文件非空（内容长度 > 0）
- [ ] 文件结构正确（包含技术栈、目录结构、相关模块等章节）

### 技术栈识别

- [ ] 编程语言已确定
- [ ] 框架已识别（含版本，如可获取）
- [ ] 架构风格已确定（单体/微服务/MVC 等）
- [ ] 构建工具已识别

### 相关模块定位

- [ ] 每个 PRD 功能（F1, F2...）都有对应的已搜索关键词和搜索结果
- [ ] 找到的相关文件有明确的功能说明和与 PRD 需求的关联
- [ ] 未遗漏关键模块（至少搜索了 routes/models/controllers/services 对应目录）

### 约定提取

- [ ] 文件命名约定已记录
- [ ] 代码组织模式已记录（从代表性文件中提取）
- [ ] 错误处理模式已记录
- [ ] 测试模式已记录

### 扩展点明确

- [ ] 需要创建的新文件有具体路径（非"在某个目录下"等模糊描述）
- [ ] 需要修改的现有文件有具体路径和修改位置
- [ ] 需要更新的配置已列出
- [ ] 数据库变更已标注（如适用）

### 依赖映射

- [ ] 上游依赖（新功能调用的现有模块）已列出
- [ ] 下游影响（受新功能影响的现有代码）已列出
- [ ] 可复用的工具函数/组件已标注文件路径

### 用户确认

- [ ] 用户确认映射结果准确（检查点通过）
