# 质量门禁：规格文档质量审查

## 产出物

`{workspace}/review-report.md`

## 目标

独立验证规格文档是否完整覆盖 PRD 需求、每个任务是否可操作、验收条件是否可测试，确保交付物质量达标。

## 使用场景

| 消费者 | 消费什么 |
|--------|----------|
| Step 4 (生成规格) | `{workspace}/review-report.md` 中的审查发现的关键问题，用于返工修复 |
| Step 6 (输出交付) | `{workspace}/review-report.md` 中的审查结论，作为交付的信心依据 |

## 验收标准

### 文件存在性

- [ ] `{workspace}/review-report.md` 文件存在且可读
- [ ] 文件非空（内容长度 > 0）
- [ ] 文件结构正确（包含概要、覆盖度分析表、问题清单）

### 审查覆盖度

- [ ] PRD 中的每条功能需求（F1, F2...）都已逐一对照
- [ ] PRD 覆盖率 ≥ 95%（允许少量标注为"范围外"的需求）
- [ ] 覆盖度分析表完整（每条需求有 covered / partial / missing 状态）

### 问题识别

- [ ] 关键问题（阻碍执行的）已全部列出，附具体文档位置和修复建议
- [ ] 改进建议（不阻碍但有优化空间）已列出
- [ ] 无遗漏的明显问题（如缺失文件路径、空白章节）

### 结论明确

- [ ] 审查结论为 PASS 或 PASS_WITH_NOTES（通过本门禁）
- [ ] 如为 NEEDS_REVISION：关键问题清单具体可操作，修复建议明确
- [ ] 如为 NEEDS_REVISION 且已返工：修复记录已附在报告中

### 报告格式

- [ ] 审查报告使用 `references/review-report-template.md` 格式
- [ ] 包含概要（质量评级、覆盖率、问题数）
- [ ] 包含覆盖度分析表
- [ ] 包含关键问题和改进建议（如有）
